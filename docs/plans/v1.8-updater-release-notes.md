# Plan v1.8.0 - Updater et Release Notes

Statut: en cours — fallback updater supprime, release notes bilingues restantes
Date: 2026-03-11

## Resume

Pour la v1.8.0, l'updater devra:

- accepter uniquement l'asset exact `AccessibleMediaConverter-Setup.exe`
- afficher les release notes en anglais si l'app est en anglais
- afficher les release notes en francais si l'app est en francais

Les releases deja publiees ne seront pas retrofitees dans cette passe.

## Decisions figees

- `v8` = `v1.8.0`
- l'asset GitHub officiel pour l'updater devient uniquement `AccessibleMediaConverter-Setup.exe`
- l'ancien fallback `AccessibleMediaConverter-Setup-<version>.exe` sera retire du code updater
- a partir de `v1.8.0`, les release notes GitHub seront bilingues dans un seul `body`
- les anciennes releases restent en l'etat
- si une ancienne release ne fournit pas de notes bilingues, fallback vers l'anglais

## Format des release notes GitHub

Le `body` GitHub d'une release a partir de `v1.8.0` devra suivre ce format:

```md
## English
<!-- AMC-RELEASE-NOTES:en:start -->
- English bullet 1
- English bullet 2
<!-- AMC-RELEASE-NOTES:en:end -->

## Francais
<!-- AMC-RELEASE-NOTES:fr:start -->
- Puce francaise 1
- Puce francaise 2
<!-- AMC-RELEASE-NOTES:fr:end -->
```

Regles:

- l'ordre public reste anglais puis francais
- l'app extrait seulement le contenu entre marqueurs
- fallback d'extraction:
  - langue demandee
  - `en`
  - `fr`
  - `body` complet si aucun marqueur valide n'est trouve
  - `No release notes provided.` si le `body` est vide

## Changement updater

Le code updater devra etre modifie ainsi:

- `find_setup_asset()` n'accepte plus que `APP_INSTALLER_FILENAME`
- suppression du fallback sur `APP_INSTALLER_BASENAME-*.exe`
- les notes cumulees doivent etre construites release par release avec extraction par langue
- la langue resolue de l'app doit etre passee explicitement a la logique updater

## Workflow release retenu

A partir de `v1.8.0`:

- stocker les notes source dans:
  - `release-notes/v1.8.0.en.md`
  - `release-notes/v1.8.0.fr.md`
- generer un fichier combine pour GitHub depuis ces deux sources
- publier la release avec `gh release create ... --notes-file <fichier combine>`
- uploader uniquement `AccessibleMediaConverter-Setup.exe`

## Validation attendue

- UI anglaise => updater et notes en anglais
- UI francaise => updater et notes en francais
- si seule une release ancienne est disponible, affichage acceptable via fallback
- `gh release view v1.8.0` ne montre qu'un seul installateur
- l'updater telecharge bien `AccessibleMediaConverter-Setup.exe`

## Bug additionnel a traiter

- Fenetre de support: la liste des types de probleme reste en anglais dans l'UI francaise.
- Cause probable deja identifiee: les libelles de `SUPPORT_ISSUE_TYPE_ITEMS` dans `core/support.py` sont traduits au runtime, mais ne sont plus presents comme entrees actives dans `locales/base.pot` et `locales/fr/LC_MESSAGES/base.po`.
- Le correctif v1.8 devra remettre ces libelles dans le flux i18n officiel, puis verifier que la liste de la fenetre support s'affiche bien en francais.
