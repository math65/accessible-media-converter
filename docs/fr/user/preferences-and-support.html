Préférences générales

  • Langue de l'application Choisissez la détection automatique, le français ou l'anglais. Changer la langue nécessite un redémarrage.
  • Dossier de sortie Choisissez le dossier source, un dossier spécifique, ou demandez un dossier à chaque batch.
  • Politique de conflit Définissez le comportement quand un fichier cible existe déjà.
  • Ouvrir le dossier final Permet d'ouvrir automatiquement le dossier de sortie à la fin d'un batch réussi ou partiellement réussi.
  • Conversions parallèles Réglez combien de tâches peuvent tourner en même temps.
  • Threads FFmpeg Fixez le nombre de threads utilisés par conversion, ou laissez en automatique.
  • Continuer après erreur Permet de laisser le batch poursuivre les autres tâches si une conversion échoue.
  • Conserver les métadonnées d'origine Conserve les tags, chapitres et la pochette intégrée du fichier source lors de la conversion et de la fusion, lorsque c'est possible. Désactivé par défaut.
  • Conserver l'arborescence des sous-dossiers d'origine Lorsque vous ajoutez un dossier contenant des sous-dossiers et convertissez vers un dossier de sortie spécifique, recrée cette arborescence de sous-dossiers sous la destination au lieu de placer tous les fichiers ensemble. Sans effet quand la sortie se fait à côté du fichier source. Désactivé par défaut.
  • Nommage des chapitres M4B Lorsque vous créez un livre audio M4B, choisissez comment nommer les chapitres générés : « Titre, sinon numéro » (titre du chapitre/fichier, sinon un numéro localisé), « Titre, sinon nom de fichier », ou « Toujours numéroté ».
  • Mises à jour automatiques Activez ou désactivez la vérification au lancement.
  • Proposer aussi les pré-versions Lorsque cette option est activée, la vérification des mises à jour prend aussi en compte les versions de test (bêta et versions candidates), pas seulement les versions stables. Cela vous permet d'essayer les nouveautés en avance. Lorsqu'une version stable est ensuite publiée, elle est aussi proposée comme mise à jour : vous ne restez donc jamais bloqué sur une version de test. Désactivé par défaut.

Mode debug

Le menu Debug active une journalisation plus détaillée et permet d'ouvrir ou d'effacer le dossier de diagnostic.

L'activation ou la désactivation du debug relance l'application pour appliquer le changement. Ce basculement n'est pas possible pendant une conversion en cours.

Activez le debug avant de reproduire un problème si vous devez ensuite envoyer un rapport plus précis au support.

Contacter le support

Le dialogue Contacter le support envoie un rapport directement au backend support. L'utilisateur remplit une adresse email, choisit un type de problème, puis décrit la situation. Une adresse email valide est obligatoire, sinon le rapport ne peut pas être envoyé.

Avant l'ouverture du formulaire, l'application vérifie s'il existe une version plus récente. Si c'est le cas, le rapport est bloqué et vous êtes invité à mettre à jour d'abord, car le problème est peut-être déjà corrigé dans la version plus récente.

Un bouton permet aussi d'afficher les détails techniques inclus dans le rapport, pour vérifier ce qui sera envoyé.

Le rapport technique reste volontairement limité : pas de chemins locaux complets, pas de noms de fichiers détaillés, pas de contenu du dossier debug envoyé automatiquement.

Accessibilité : chaque champ est accessible au clavier avec la touche Tab, et les dialogues sont lus par NVDA.

Si l'envoi automatique échoue, le dialogue propose un mode de repli avec copie de l'adresse, du sujet et du rapport complet pour un envoi manuel.

Mises à jour

L'application sait vérifier les releases GitHub et afficher les notes des versions plus récentes que celle installée.

  • vérification manuelle depuis le menu Aide ;
  • vérification automatique au lancement si l'option est activée ;
  • installation via le setup officiel téléchargé localement.

Une fois l'installeur téléchargé, l'application le vérifie (taille du fichier et empreinte SHA-256) avant de l'exécuter. La mise à jour s'installe ensuite silencieusement avec une barre de progression, puis l'application se relance automatiquement.
