Preferences, support, and updates

This page groups everything that does not belong to a single conversion: global preferences, debug mode, support reports, and application updates.

On this page

  • Debug mode

General preferences

  • Application language Choose automatic detection, French, or English. Changing the language requires a restart.
  • Output folder Choose the source folder, a specific folder, or ask for a folder at each batch.
  • Conflict policy Define what happens when an output file already exists.
  • Open output folder Automatically opens the output folder at the end of a successful or partially successful batch.
  • Parallel conversions Set how many tasks may run at the same time.
  • FFmpeg threads Set the number of threads used per conversion, or keep automatic mode.
  • Continue after error Lets the batch continue with other tasks if one conversion fails.
  • Automatic update checks Enable or disable update checks at startup.

Debug mode

The Debug menu enables more detailed logging and lets you open or clear the diagnostic folder.

Enabling or disabling debug mode restarts the application to apply the change. This switch is not available during an active conversion.

Enable debug mode before reproducing a problem if you may later need to send a more precise report to support.

Contact support

The Contact Support dialog sends a report directly through the support backend. The user enters an email address, chooses a problem type, and describes the situation.

A button can also display the technical details included in the report, so you can review what will be sent.

The technical report remains intentionally limited: no full local paths, no detailed file names, and no automatic upload of the debug folder contents.

If automatic sending fails, the dialog provides a fallback with the support address, the subject, and the full report ready to copy for manual sending.

Updates

The application can check GitHub releases and display notes for versions that are newer than the installed one.

  • manual check from the Help menu;
  • automatic check at startup if enabled;
  • installation through the official setup downloaded locally.
