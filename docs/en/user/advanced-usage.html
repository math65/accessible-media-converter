Advanced usage

This page groups the more advanced features: merging files (including chaptered M4B audiobooks), editing metadata with auto-detection from file names, detailed video settings, output audio codecs, track management, parallel batches, debug mode, and support.

Merging files

You can merge two or more files into a single output (FFmpeg concat). When the target is the M4B format (audiobook), the merge produces one chapter per source file. Converting a single already-chaptered file to M4B preserves its chapters, and the .m4b file is also accepted as input. The default is stereo at 128 kb/s, but everything is adjustable (mono / 64 kb/s works well for voice). Select at least two files (Ctrl+click, Shift+click, or Ctrl+A), then start the merge.

Editing metadata

Right-click an audio or video file, or several (or the Menu/Applications key, or Shift+F10, on the selected row), then Edit Metadata...: standard tags (title, artist, album, album artist, composer, year, track, disc, genre, comment) and the cover art (replace from JPEG/PNG, remove, keep). Target: apply during conversion or re-tag the original now. The fields you fill in can be applied to a whole batch.

For a video, a Content type selector (Movie / TV Show / Other) adapts the fields (Movie: synopsis; TV Show: series, season, episode, episode title). Audio files also offer Grouping, Copyright, and a multi-line Lyrics field.

Auto-detection from the file name

The editor can fill some fields from the file name, offered through a checkbox: for a video it reads the season and episode (SxxExx pattern), and for audio it reads the track number. Leave the box unchecked to keep the fields untouched.

Advanced video settings

For MP4 and MKV outputs, video can be copied or re-encoded. In re-encode mode, you get a main preset and then advanced settings.

The Video Preset is a global shortcut. It applies several consistent values together, including CRF, the Encoder Preset, the H.264 profile, and the pixel format. The Encoder Preset itself only controls one trade-off: encoding speed versus compression efficiency.

  • CRF to control target quality;
  • x264 preset to trade speed against compression;
  • H.264 profile for compatibility;
  • pixel format for rendering and playback.
If you manually change the Encoder Preset or another advanced setting, you are customizing the video preset that was selected first.

Advanced audio settings

Audio inside a video output. In a video output, audio settings stay separate from video settings. You can choose a codec compatible with the container, then adjust bitrate, quality, channels, sample rate, and normalization depending on the codec.

  • MP4: AAC or MP3
  • MKV: AAC, MP3, Opus, or FLAC

Audio as the main output format. When the output is an audio format, you can pick from MP3, AAC/M4A, WAV, FLAC, ALAC, OGG/Vorbis, WMA, and M4B.

  • Stream copy to remux without re-encoding when the source codec matches;
  • CBR or VBR bitrate mode depending on the codec;
  • bitrate, sample rate, and channels (for example mono / stereo);
  • normalization to a target of -16 LUFS;
  • FLAC compression level and bit depth for lossless output.

Track management

The track manager lets you choose which video, audio, and subtitle tracks to keep, change their order, and adjust some metadata and dispositions.

In practice, these actions open from a right-click on a video file in the list (right-click, or the Menu/Applications key, or Shift+F10, on the selected row). Manage Tracks... is offered for an MP4 or MKV output, while Choose Audio Track... appears for audio extraction when several tracks exist.

For video-to-audio conversion, the application extracts only one audio track.

Batch and performance

The application can run several conversions in parallel. The preferences let you define:

  • the maximum number of simultaneous conversions;
  • the conflict policy for output files;
  • the number of FFmpeg threads per conversion.

Debug, support, and updates

If a problem must be reported, you can enable debug mode, reproduce the issue, and then open the integrated support dialog. The form sends a clean report without exposing full local paths.

The support dialog can also show the included technical details and provide a manual copy fallback if automatic sending fails. Reporting a problem requires a valid email address, and the report is blocked if your version is out of date (a fresh update check runs first, since the bug may already be fixed).

Updates can also be checked manually from the Help menu, or automatically at startup if the option is enabled. Before running, the downloaded installer is verified for integrity (file size and SHA-256 checksum). After a silent update, the application relaunches itself automatically.

Other preferences worth noting:

  • M4B chapter naming: title otherwise a localized number / title otherwise the file name / always numbered;
  • Preserve original metadata: keep the source tags during conversion.
